# نظام إدارة المستخدمين والصلاحيات - صحيفة سبق

## 📋 نظرة عامة

نظام متقدم وشامل لإدارة المستخدمين والصلاحيات في صحيفة سبق الإلكترونية، يوفر تحكمًا دقيقًا في الصلاحيات والأدوار مع قابلية التوسع المستقبلية.

## 🏗️ البنية التقنية

### قاعدة البيانات

#### الجداول الرئيسية:

1. **users** - المستخدمين
2. **roles** - الأدوار
3. **permissions** - الصلاحيات
4. **role_permissions** - ربط الأدوار بالصلاحيات
5. **user_permissions** - صلاحيات إضافية/استثنائية للمستخدمين
6. **user_sections** - ربط المستخدمين بالأقسام
7. **user_activity_logs** - سجل النشاطات
8. **user_sessions** - جلسات المستخدمين
9. **invitations** - الدعوات
10. **user_performance** - تقارير الأداء

### API Endpoints

#### المستخدمين
- `GET /api/users` - قائمة المستخدمين
- `POST /api/users` - إنشاء مستخدم جديد
- `GET /api/users/[id]` - تفاصيل مستخدم
- `PUT /api/users/[id]` - تحديث مستخدم
- `DELETE /api/users/[id]` - حذف مستخدم
- `PATCH /api/users/[id]/suspend` - تعليق/تفعيل مستخدم

#### الأدوار
- `GET /api/roles` - قائمة الأدوار
- `POST /api/roles` - إنشاء دور جديد
- `GET /api/roles/[id]` - تفاصيل دور
- `PUT /api/roles/[id]` - تحديث دور
- `DELETE /api/roles/[id]` - حذف دور

#### الصلاحيات
- `GET /api/permissions` - قائمة جميع الصلاحيات المتاحة

#### النشاطات
- `GET /api/activities` - سجل النشاطات مع فلترة

## 👥 الأدوار المتاحة

### أدوار النظام الأساسية (لا يمكن حذفها):

1. **مشرف عام** - صلاحيات كاملة

3. **نائب رئيس تحرير** - مساعدة رئيس التحرير
4. **محرر** - كتابة وتحرير المقالات
5. **محرر أقسام** - إدارة قسم معين
6. **مراسل** - جمع الأخبار وكتابة التقارير
7. **مدقق لغوي** - تدقيق المحتوى
8. **مدير وسائط** - إدارة الصور والفيديوهات
9. **محلل بيانات** - تحليل الأداء
10. **مشاهد** - مشاهدة فقط

## 🎯 المميزات الرئيسية

### 1. إدارة المستخدمين
- إنشاء مستخدمين يدوياً أو عبر دعوة
- تعيين أدوار وصلاحيات
- ربط بالأقسام المسموح بها
- تتبع حالة المستخدم (نشط/معلق/معلق)
- سجل دخول مفصل

### 2. نظام الأدوار المرن
- أدوار نظام محمية
- إمكانية إنشاء أدوار مخصصة
- وصف واضح لكل دور
- عدد المستخدمين والصلاحيات

### 3. صلاحيات دقيقة (Granular)
- 38 صلاحية مختلفة
- مجمعة في 5 فئات
- صلاحيات إضافية/استثنائية
- صلاحيات مؤقتة (قادمة)

### 4. سجل النشاطات الشامل
- تسجيل جميع الإجراءات
- معلومات المستخدم والوقت
- عنوان IP والجهاز
- فلترة وبحث متقدم

### 5. الأمان والحماية
- تشفير كلمات المرور
- جلسات آمنة
- تتبع محاولات الدخول
- دعم المصادقة الثنائية (قادم)

## 🚀 خطوات التشغيل

### 1. إعداد قاعدة البيانات
```bash
# تنفيذ ملف إنشاء الجداول
psql -U postgres -d sabq_db -f database/schema.sql

# إدراج البيانات الأولية
psql -U postgres -d sabq_db -f database/seed.sql
```

### 2. إعداد متغيرات البيئة
```env
DATABASE_URL=postgresql://user:password@localhost:5432/sabq_db
JWT_SECRET=your-secret-key
NEXTAUTH_URL=http://localhost:3000
```

### 3. تشغيل المشروع
```bash
cd frontend
npm install
npm run dev
```

### 4. الوصول للصفحات
- إدارة الفريق: http://localhost:3000/dashboard/team
- الأدوار: http://localhost:3000/dashboard/roles
- النشاطات: http://localhost:3000/dashboard/activities

## 🔧 التخصيص والتوسع

### إضافة صلاحية جديدة:
1. إضافتها في جدول `permissions`
2. تحديث API endpoint
3. إضافتها في واجهة المستخدم

### إضافة دور جديد:
1. استخدام واجهة إنشاء الأدوار
2. تحديد الصلاحيات المطلوبة
3. ربط المستخدمين بالدور

### تخصيص النشاطات:
1. إضافة أنواع جديدة في `ACTIVITY_ACTIONS`
2. تحديث دالة `logActivity`
3. إضافة الترجمة في الواجهة

## 📝 ملاحظات مهمة

1. **الأمان**: تأكد من تشفير الاتصال HTTPS في الإنتاج

3. **السجلات**: احتفظ بسجلات النشاط لمدة كافية (90 يوم مثلاً)
4. **الأداء**: أضف فهارس إضافية عند نمو البيانات
